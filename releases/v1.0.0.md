## Overview

`xk6-subcommand-explore` is a k6 extension that adds an `explore` subcommand to help discover available k6 extensions from the official extension registry. It's designed to work with k6's Automatic Extension Resolution feature, allowing you to quickly find compatible extensions, filter them by type or tier, and view detailed information.

The extension provides both human-readable table output and machine-readable JSON output, making it suitable for both interactive use and CI/CD automation.

## Usage

**List all extensions:**
```bash
k6 x explore
```

**Show detailed information with repository URLs:**
```bash
k6 x explore --detailed
```

**Show compact view:**
```bash
k6 x explore --brief
```

**Show full descriptions without truncation:**
```bash
k6 x explore --no-trunc
```

**Output as JSON:**
```bash
k6 x explore --json
```

**Filter by type:**
```bash
k6 x explore --type javascript
```

**Filter by tier:**
```bash
k6 x explore --tier official
```

**Combine filters:**
```bash
k6 x explore --tier official --type javascript
```

**Process with jq:**
```bash
k6 x explore --json | jq '.[] | select(.tier == "official")'
```

## Build

Currently, you need to build a custom k6 binary with this extension to use the `explore` subcommand. Use the [xk6](https://github.com/grafana/xk6) tool:

```bash
xk6 build --with github.com/grafana/xk6-subcommand-explore@v1.0.0
```

In the near future, k6 will introduce subcommand support to the Automatic Extension Resolution feature, making this extension available automatically without requiring a custom build.
